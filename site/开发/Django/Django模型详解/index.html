    Django模型详解

    Model (模型) 简而言之即数据模型，是一个Django应用的核心。模型不是数据本身（比如数据表里的数据), 而是抽象的描述数据的构成和逻辑关系。

    每个Django的模型(model)实际上是个类，继承了<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">models.Model</font>。每个Model应该包括属性(字段)，关系（比如单对单，单对多和多对多)和方法。当你定义好Model模型后，Django的接口会自动帮你在数据库生成相应的数据表(table)。这样你就不用自己用SQL语言创建表格或在数据库里操作创建表格了，是不是很省心？

    模型定义小案例¶

    假设你要开发一个名叫<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">bookstore</font>的应用，专门来管理书店里的书籍。我们首先要为书本和出版社创建模型。出版社有名字和地址。书有名字，描述和添加日期。我们还需要利用ForeignKey定义了出版社与书本之间单对多的关系，因为一个出版社可以出版很多书，每本书都有对应的出版社。我们定义了<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">Publisher</font>和<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">Book</font>模型，它们都继承了<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">models.Model</font>。你能看出代码有什么问题吗?

    # models.py
from django.db import models

class Publisher(models.Model):
    name = models.CharField(max_length=30)
    address = models.CharField()

    def __str__(self):
        return self.name

class Book(models.Model):
    name = models.CharField(max_length=30)
    description = models.TextField(blank=True, null=True)
    publisher = ForeignKey(Publisher)
    add_date = models.DateField()

    def __str__(self):
        return self.name

    模型创建好后，当你运行<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">python manage.py migrate</font> 命令创建数据表的时候你会遇到错误，错误原因如下：

    • <font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">CharField</font>里的<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">max_length</font>选项没有定义
    • <font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">ForeignKey(Publisher)</font>里的<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">on_delete</font>选项有没有定义

    所以当你定义Django模型Model的时候，你一定要十分清楚2件事:

    • 这个Field是否有必选项, 比如<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">CharField</font>的<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">max_length</font>和<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">ForeignKey</font>的<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">on_delete</font>选项是必须要设置的。
    • 这个Field是否必需(blank = True or False)，是否可以为空 (null = True or False)。这关系到数据的完整性。

    # models.py
from django.db import models

class Publisher(models.Model):
    name = models.CharField(max_length=30)
    address = models.CharField(max_length=60)

    def __str__(self):
        return self.name

class Book(models.Model):
    name = models.CharField(max_length=30)
    description = models.TextField(blank=True, default='')
    publisher = ForeignKey(Publisher,on_delete=models.CASCADE)
    add_date = models.DateField(auto_now_add=True)

    def __str__(self):
        return self.name

    修改模型后，你需要连续运行<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">python manage.py makemigrations</font>和<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">python manage.py migrate</font>这两个命令，前者检查模型有无变化，后者将变化迁移至数据表。如果一切顺利，Django会在数据库(默认sqlite)中生成或变更由<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">appname_modelname</font>组成的数据表，本例两张数据表分别为<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">bookstore_publisher</font>和<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">bookstore_book</font>。

    模型的组成¶

    一个标准的Django模型分别由模型字段、META选项和方法三部分组成。我们接下来对各部分进行详细介绍。Django官方编码规范建议按如下方式排列：

    • 定义的模型字段：包括基础字段和关系字段
    • 自定义的Manager方法：改变模型
    • <font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">class Meta选项</font>: 包括排序、索引等等(可选)。
    • <font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">def __str__()</font>：定义单个模型实例对象的名字(可选)。
    • <font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">def save()</font>：重写save方法(可选)。
    • <font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">def get_absolute_url()</font>：为单个模型实例对象生成独一无二的url(可选)
    • 其它自定义的方法。

    模型的字段¶

    <font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">models.Model</font>提供的常用模型字段包括基础字段和关系字段。

    基础字段¶

    CharField()

    一般需要通过max_length = xxx 设置最大字符长度。如不是必填项，可设置blank = True和default = ‘‘。如果用于username, 想使其唯一，可以设置<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">unique = True</font>。如果有choice选项，可以设置 choices = XXX_CHOICES

    TextField()

    适合大量文本，max_length = xxx选项可选。

    DateField() 和DateTimeField()

    可通过default=xx选项设置默认日期和时间。

    • 对于DateTimeField: default=timezone.now - 先要<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">from django.utils import timezone</font>
    • 如果希望自动记录一次修改日期(modified)，可以设置: <font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">auto_now=True</font>
    • 如果希望自动记录创建日期(created),可以设置<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">auto_now_add=True</font>

    EmailField()

    如不是必填项，可设置blank = True和default = ‘。一般Email用于用户名应该是唯一的，建议设置unique = True

    IntegerField(), SlugField(), URLField()，BooleanField()

    可以设置blank = True or null = True。对于BooleanField一般建议设置<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">defaut = True or False</font>

    FileField(upload_to=None, max_length=100) - 文件字段

    • upload_to = “/some folder/”：上传文件夹路径
    • max_length = xxxx：文件最大长度

    ImageField (upload_to=None, max_length=100,)- 图片字段

    • upload_to = “/some folder/”: 指定上传图片路径

    关系字段¶

    OneToOneField(to, on_delete=xxx, options) - 单对单关系

    • to必需指向其他模型，比如 Book or ‘self’ .
    • 必需指定<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">on_delete</font>选项(删除选项): i.e, “<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">on_delete = models.CASCADE</font>” or “<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">on_delete = models.SET_NULL</font>” .
    • 可以设置 “<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">related_name = xxx</font>” 便于反向查询。

    ForeignKey(to, on_delete=xxx, options) - 单对多关系

    • to必需指向其他模型，比如 Book or ‘self’ .
    • 必需指定<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">on_delete</font>选项(删除选项): i.e, “<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">on_delete = models.CASCADE</font>” or “<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">on_delete = models.SET_NULL</font>” .
    • 可以设置”default = xxx” or “null = True” ;
    • 如果有必要，可以设置 “<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">limit_choices_to =</font> “,
    • 可以设置 “<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">related_name = xxx</font>” 便于反向查询。

    ManyToManyField(to, options) - 多对多关系

    • to 必需指向其他模型，比如 User or ‘self’ .
    • 设置 “<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">symmetrical = False</font> “ 表示多对多关系不是对称的，比如A关注B不代表B关注A
    • 设置 “<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">through = 'intermediary model'</font> “ 如果需要建立中间模型来搜集更多信息。
    • 可以设置 “<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">related_name = xxx</font>” 便于反向查询。

    from django.db import models

class Person(models.Model):
    name = models.CharField(max_length=128)

    def __str__(self):
        return self.name

class Group(models.Model):
    name = models.CharField(max_length=128)
    members = models.ManyToManyField(Person, through='Membership')

    def __str__(self):
        return self.name

class Membership(models.Model):
    person = models.ForeignKey(Person, on_delete=models.CASCADE)
    group = models.ForeignKey(Group, on_delete=models.CASCADE)
    date_joined = models.DateField()
    invite_reason = models.CharField(max_length=64)

    对于<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">OneToOneField</font>和<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">ForeignKey</font>, <font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">on_delete</font>选项和<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">related_name</font>是两个非常重要的设置，前者决定了了关联外键删除方式，后者决定了模型反向查询的名字。

    on_delete删除选项¶

    Django提供了如下几种关联外键删除选项, 可以根据实际需求使用。

    • <font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">CASCADE</font>：级联删除。当你删除publisher记录时，与之关联的所有 book 都会被删除。
    • <font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">PROTECT</font>: 保护模式。如果有外键关联，就不允许删除，删除的时候会抛出ProtectedError错误，除非先把关联了外键的记录删除掉。例如想要删除publisher，那你要把所有关联了该publisher的book全部删除才可能删publisher。
    • <font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">SET_NULL</font>: 置空模式。删除的时候，外键字段会被设置为空。删除publisher后，book 记录里面的publisher_id 就置为null了。
    • <font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">SET_DEFAULT</font>: 置默认值，删除的时候，外键字段设置为默认值。
    • <font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">SET()</font>: 自定义一个值。
    • <font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">DO_NOTHING</font>：什么也不做。删除不报任何错，外键值依然保留，但是无法用这个外键去做查询。

    related_name选项¶

    <font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">related_name</font>用于设置模型反向查询的名字，非常有用。在文初的<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">Publisher</font>和<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">Book</font>模型里，我们可以通过<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">book.publisher</font>获取每本书的出版商信息，这是因为<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">Book</font>模型里有<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">publisher</font>这个字段。但是<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">Publisher</font>模型里并没有<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">book</font>这个字段，那么我们如何通过出版商反查其出版的所有书籍信息呢？

    Django对于关联字段默认使用<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">模型名_set</font>进行反查，即通过<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">publisher.book_set.all</font>查询。但是<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">book_set</font>并不是一个很友好的名字，我们更希望通过<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">publisher.books</font>获取一个出版社已出版的所有书籍信息，这时我们就要修改我们的模型了，将<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">related_name</font>设为<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">books</font>, 如下所示：

from django.db import models

class Publisher(models.Model):
    name = models.CharField(max_length=30)
    address = models.CharField(max_length=60)

    模型的META选项¶

    • <font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">abstract=True</font>: 指定该模型为抽象模型
    • <font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">proxy=True</font>: 指定该模型为代理模型
    • <font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">verbose_name=xxx</font>和<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">verbose_name_plural=xxx</font>: 为模型设置便于人类阅读的别名
    • <font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">db_table= xxx</font>: 自定义数据表名
    • <font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">odering=['-pub-date']</font>: 自定义按哪个字段排序，<font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">-</font>代表逆序
    • <font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">permissions=[]</font>: 为模型自定义权限
    • <font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">managed=False</font>: 默认为True，如果为False，Django不会为这个模型生成数据表
    • <font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">indexes=[]</font>: 为数据表设置索引，对于频繁查询的字段，建议设置索引
    • <font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">constraints=</font>: 给数据库中的数据表增加约束。

    模型的方法¶

    标准方法¶

    以下三个方法是Django模型自带的三个标准方法：

    • <font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">def __str__()</font>：给单个模型对象实例设置人为可读的名字(可选)。
    • <font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">def save()</font>：重写save方法(可选)。
    • <font style="color:rgb(92, 89, 98);background-color:rgb(245, 246, 250);">def get_absolute_url()</font>：为单个模型实例对象生成独一无二的url(可选)
