    入门¶

    要求¶

    • Python（3.8，3.9，3.10，3.11）
    • Django（3.2，4.0，4.1，4.2）
    • Django REST框架（3.10，3.11，3.12，3.13，3.14）

    这些是官方支持的python和包版本。 其他版本可能会工作。 您可以自由修改tox配置并看看有什么可能。

    安装¶

    简单的JWT可以通过pip安装：

    Python
    1
    pip install djangorestframework-simplejwt

    加密验证（可选）¶

    如果您计划使用某些数字签名算法（即RSA和ECDSA;访问PyJWT了解其他算法）对令牌进行编码或解码，则需要安装加密库。这可以显式安装，也可以作为<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">djangorestframework-simplejwt</font>需求中的必需附加项安装：

    Python
    1
    pip install djangorestframework-simplejwt[crypto]

    在使用<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">djangorestframework-simplejwt[crypto]</font>的项目中，建议在需求文件中使用<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">Simple</font><font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);"> </font><font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">JWT</font>格式，因为单独的<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">cryptography</font>需求行可能会被误认为未使用的需求并被删除。

    <font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">ACCESS_TOKEN_LIFETIME</font>¶

    一个<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">datetime.timedelta</font>对象，指定访问令牌的有效期。 该<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">timedelta</font>值在令牌生成期间添加到当前UTC时间，以获取令牌的默认“ext”声明值。

    <font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">REFRESH_TOKEN_LIFETIME</font>¶

    一个<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">datetime.timedelta</font>对象，指定刷新令牌的有效期。 在令牌生成期间，此<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">timedelta</font>值被添加到当前UTC时间，以获得令牌的默认“exp”声明值。

    <font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">ROTATE_REFRESH_TOKENS</font>¶

    当设置为<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">True</font>时，如果向<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">TokenRefreshView</font>提交刷新令牌，则新的刷新令牌将与新的访问令牌一起沿着返回。 这个新的刷新令牌将通过JSON响应中的“refresh”键提供。 新的刷新令牌将具有更新的到期时间，该到期时间通过将<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">REFRESH_TOKEN_LIFETIME</font>设置中的时间增量添加到发出请求的当前时间来确定。 如果正在使用黑名单应用程序，并且<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">BLACKLIST_AFTER_ROTATION</font>设置设置为<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">True</font>，则提交到刷新视图的刷新令牌将添加到黑名单中。

    <font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">BLACKLIST_AFTER_ROTATION</font>¶

    当设置为<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">True</font>时，如果黑名单应用程序正在使用且<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">TokenRefreshView</font>设置设置为<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">ROTATE_REFRESH_TOKENS</font>，则会将提交给<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">True</font>的刷新令牌添加到黑名单中。您需要在设置文件中将<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">'rest_framework_simplejwt.token_blacklist',</font>添加到<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">INSTALLED_APPS</font>中才能使用此设置。

    了解更多关于黑名单应用程序。

    <font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">UPDATE_LAST_LOGIN</font>¶

    当设置为<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">True</font>时，auth_user表中的last_login字段在登录时更新（TokenObtainPairView）。

    警告：更新last_login将显著增加数据库事务的数量。滥用视图的人可能会降低服务器的速度，这可能是一个安全漏洞。如果你真的想这样做，至少用DRF节流端点。

    <font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">ALGORITHM</font>¶

    PyJWT库中的算法，将用于对令牌执行签名/验证操作。 为了使用对称HMAC签名和验证，可以使用以下算法：<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">'HS256'</font>、<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">'HS384'</font>、<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">'HS512'</font>。 当选择HMAC算法时，<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">SIGNING_KEY</font>设置将同时用作签名密钥和验证密钥。 在这种情况下，将忽略<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">VERIFYING_KEY</font>设置。 为了使用非对称RSA签名和验证，可以使用以下算法：<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">'RS256'</font>、<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">'RS384'</font>、<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">'RS512'</font>。 选择RSA算法时，必须将<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">SIGNING_KEY</font>设置设置为包含RSA私钥的字符串。 同样，<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">VERIFYING_KEY</font>设置必须设置为包含RSA公钥的字符串。

    <font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">SIGNING_KEY</font>¶

    用于对生成的令牌的内容进行签名的签名密钥。 对于HMAC签名，这应该是一个随机字符串，其中至少包含签名协议所需的数据位数。 对于RSA签名，这应该是一个包含2048位或更长的RSA私钥的字符串。 由于Simple JWT默认使用256位HMAC签名，因此<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">SIGNING_KEY</font>设置默认为django项目的<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">SECRET_KEY</font>设置值。 虽然这是Simple JWT可以提供的最合理的默认值，但建议开发人员将此设置更改为独立于django项目密钥的值。 这将使更改用于令牌的签名密钥变得更容易，如果它被泄露的话。

    <font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">VERIFYING_KEY</font>¶

    用于验证生成的令牌内容的验证密钥。 如果HMAC算法已由<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">ALGORITHM</font>设置指定，则将忽略<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">VERIFYING_KEY</font>设置，并使用<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">SIGNING_KEY</font>设置的值。 如果已通过<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">ALGORITHM</font>设置指定RSA算法，则必须将<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">VERIFYING_KEY</font>设置设置为包含RSA公钥的字符串。

    <font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">AUDIENCE</font>¶

    受众声称被包括在生成的令牌中和/或在解码的令牌中被验证。当设置为<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">None</font>时，此字段将从令牌中排除，并且不进行验证。

    <font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">ISSUER</font>¶

    发行方声称被包括在生成的令牌中和/或在解码的令牌中被验证。当设置为<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">None</font>时，此字段将从令牌中排除，并且不进行验证。

    <font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">JWK_URL</font>¶

    JWK_URL用于动态解析验证令牌签名所需的公钥。例如，当使用Auth 0时，您可以将其设置为'https：//yourdomain.auth0.com/.well-known/jwks.json'。当设置为<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">None</font>时，此字段将从令牌后端中排除，并且在验证期间不使用。

    <font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">LEEWAY</font>¶

    Leeway用于为到期时间提供一定的余量。这可以是一个整数秒或<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">datetime.timedelta</font>。更多信息请参考https：//pyjwt.readthedocs.io/en/latest/usage.html#application-time-claim-exp。

    <font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">AUTH_HEADER_TYPES</font>¶

    需要身份验证的视图将接受的授权标头类型。 例如，值<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">'Bearer'</font>意味着需要身份验证的视图将查找具有以下格式的头：<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">Authorization:</font><font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);"> </font><font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">Bearer</font><font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);"> </font><font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);"><token></font>。 此设置还可以包含可能的报头类型的列表或元组（例如<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">('Bearer',</font><font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);"> </font><font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">'JWT')</font>）。 如果以这种方式使用列表或元组，并且身份验证失败，则集合中的第一项将用于在响应中构建“WWW-Authenticate”标头。

    <font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">AUTH_HEADER_NAME</font>¶

    用于身份验证的授权标头名称。默认值是<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">HTTP_AUTHORIZATION</font>，它将接受请求中的<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">Authorization</font>头。例如，如果您想在请求的标题中使用<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">X_Access_Token</font>，请在设置中将<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">AUTH_HEADER_NAME</font>指定为<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">HTTP_X_ACCESS_TOKEN</font>。

    <font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">USER_ID_FIELD</font>¶

    用户模型中的数据库字段，将包含在生成的令牌中以标识用户。 建议此设置的值指定一个字段，该字段在选定初始值后通常不会更改。 例如，指定“用户名”或“电子邮件”字段将是一个糟糕的选择，因为帐户的用户名或电子邮件可能会根据给定服务中的帐户管理的设计方式而更改。 这可以允许使用旧用户名创建新帐户，而现有令牌仍然有效，使用该用户名作为用户标识符。

    <font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">USER_ID_CLAIM</font>¶

    生成的令牌中的声明将用于存储用户标识符。例如，设置值<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">'user_id'</font>意味着生成的令牌包括包含用户标识符的“user_id”声明。

    <font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">USER_AUTHENTICATION_RULE</font>¶

    可调用以确定是否允许用户进行身份验证。此规则在处理有效令牌后应用。用户对象作为参数传递给可调用对象。默认规则是检查<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">is_active</font>标志是否仍为<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">True</font>。可调用对象必须返回一个布尔值，如果已授权，则返回<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">True</font>，否则返回<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">False</font>，结果是一个401状态代码。

    <font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">AUTH_TOKEN_CLASSES</font>¶

    指向类的点路径列表，这些类指定允许证明身份验证的令牌类型。 更多关于这一点在下面的“令牌类型”部分。

    <font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">TOKEN_TYPE_CLAIM</font>¶

    用于存储令牌类型的声明名称。 更多关于这一点在下面的“令牌类型”部分。

    <font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">JTI_CLAIM</font>¶

    用于存储令牌的唯一标识符的声明名称。 此标识符用于标识黑名单应用中已撤销的令牌。在某些情况下，可能需要使用默认“jti”声明之外的另一个声明来存储此类值。

    <font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">TOKEN_USER_CLASS</font>¶

    由经过验证的令牌支持的无状态用户对象。仅用于JWTStatelessUserAuthentication身份验证后端。该值是指向子类<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">rest_framework_simplejwt.models.TokenUser</font>的虚线路径，这也是默认值。

    <font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">SLIDING_TOKEN_LIFETIME</font>¶

    一个<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">datetime.timedelta</font>对象，指定滑动令牌在多长时间内有效以证明身份验证。 在令牌生成期间，此<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">timedelta</font>值被添加到当前UTC时间，以获得令牌的默认“exp”声明值。 更多关于这一点在“滑动令牌”下面的部分.

    <font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">SLIDING_TOKEN_REFRESH_LIFETIME</font>¶

    一个<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">datetime.timedelta</font>对象，指定滑动令牌的刷新有效期。 该<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">timedelta</font>值在令牌生成期间添加到当前UTC时间，以获取令牌的默认“ext”声明值。更多关于这一点在"滑动令牌"下面的部分.

    <font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">SLIDING_TOKEN_REFRESH_EXP_CLAIM</font>¶

    声明名称，用于存储滑动令牌刷新周期的到期时间。 更多关于这一点在“滑动令牌”下面的部分.

    <font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">CHECK_REVOKE_TOKEN</font>¶

    如果此字段设置为<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">True</font>，系统将通过比较用户当前密码的md5哈希值与JWT令牌有效负载内的REVOKE_TOKEN_CLAIM字段中存储的值来验证令牌是否已被撤销。

    <font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">REVOKE_TOKEN_CLAIM</font>¶

    用于存储用户哈希密码的声明名称。如果该REVOKE_TOKEN字段的值是<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">True</font>，则该字段将被包括在JWT有效载荷中。

    令牌类型¶

    Simple JWT提供了两种不同的令牌类型，可用于证明身份验证。 在令牌的有效负载中，其类型可以由其令牌类型声明的值标识，默认情况下为“token_type”。 这可能具有“访问”、“滑动”或“刷新”的值，但是刷新令牌此时不被认为对认证有效。 用于存储类型的声明名称可以通过更改<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">TOKEN_TYPE_CLAIM</font>设置来自定义。

    默认情况下，Simple JWT需要一个“访问”令牌来证明身份验证。 允许的授权令牌类型由<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">AUTH_TOKEN_CLASSES</font>设置的值确定。 此设置包含指向标记类的点路径列表。 默认情况下，它包含<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">'rest_framework_simplejwt.tokens.AccessToken'</font>点路径，但也可能包含<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">'rest_framework_simplejwt.tokens.SlidingToken'</font>点路径。这两个点路径中的一个或两个都可能出现在auth token类列表中。 如果它们都存在，则这两种令牌类型都可以用于证明认证。

    滑动令牌¶

    滑动令牌为令牌的用户提供了更方便的体验，其代价是安全性较低，并且在使用黑名单应用程序的情况下，性能较低。 滑动令牌是同时包含过期声明和刷新过期声明的令牌。 只要滑动令牌的过期声明中的时间戳尚未过期，它就可以用来证明身份验证。 此外，只要它的刷新过期声明中的时间戳还没有过去，它也可以被提交到刷新视图，以获得具有更新的过期声明的它自己的另一个副本。

    如果您想使用滑动令牌，请将<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">AUTH_TOKEN_CLASSES</font>设置更改为<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">('rest_framework_simplejwt.tokens.SlidingToken',)</font>。 （或者，如果您希望允许两种令牌类型都用于身份验证，则<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">AUTH_TOKEN_CLASSES</font>设置可以包括指向<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">AccessToken</font>模块中的<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">SlidingToken</font>和<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">rest_framework_simplejwt.tokens</font>令牌类的点路径。）

    此外，在访问令牌特定的<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">TokenObtainSlidingView</font>和<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">TokenRefreshSlidingView</font>视图的url旁边或代替访问令牌特定的<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">TokenObtainPairView</font>和<font style="color:rgb(231, 76, 60);background-color:rgb(252, 252, 252);">TokenRefreshView</font>视图的url，包括用于滑动令牌特定的url：
