    Pyproject.toml配置文件

    1. 理解 **<font style="color:rgb(64, 64, 64);">pyproject.toml</font>** 的核心作用¶

    • PEP 621 标准化：<font style="color:rgb(64, 64, 64);">pyproject.toml</font> 是 Python 社区推出的配置文件格式，旨在统一项目元数据（如名称、版本、依赖项）和构建系统的配置（如构建工具、插件）。
    • 替代传统文件：逐渐取代 <font style="color:rgb(64, 64, 64);">setup.py</font>、<font style="color:rgb(64, 64, 64);">setup.cfg</font>、<font style="color:rgb(64, 64, 64);">requirements.txt</font> 等文件，成为现代 Python 项目的核心配置。

    2. 学习 **<font style="color:rgb(64, 64, 64);">pyproject.toml</font>** 的通用结构¶

    基础部分（PEP 621 标准）：

    TOML
    1
2
3
4
5
    [project]
name = "my_project"
version = "0.1.0"
dependencies = ["requests>=2.25"]
authors = [{name = "Your Name", email = "you@example.com"}]

    构建系统（定义构建工具）：

    TOML
    1
2
3
    [build-system]
requires = ["setuptools>=42", "wheel"]
build-backend = "setuptools.build_meta"

    工具扩展配置（如 Poetry、PDM、Black 等工具的专用配置）：

    TOML
    1
2
3
    [tool.poetry]
[tool.pdm]
[tool.uv]

    3. 工具间的配置异同¶

    工具 特点 配置兼容性
    Poetry 专注于依赖管理和打包，强锁定机制（<font style="color:rgb(64, 64, 64);">poetry.lock</font>
    ）    		 使用 <font style="color:rgb(64, 64, 64);">[tool.poetry]</font>
    定义元数据和依赖，不完全兼容 PEP 621，需手动对齐。
    PDM 支持 PEP 621，依赖解析快，使用 <font style="color:rgb(64, 64, 64);">__pypackages__</font>
    目录    		 完全兼容 PEP 621，配置在 <font style="color:rgb(64, 64, 64);">[project]</font>
    下，部分扩展在 <font style="color:rgb(64, 64, 64);">[tool.pdm]</font>
    。
    UV 由 Astral 开发，极速的依赖安装器，兼容 <font style="color:rgb(64, 64, 64);">pip</font>
    和 <font style="color:rgb(64, 64, 64);">pip-tools</font>    		 主要作为安装器，不直接管理 <font style="color:rgb(64, 64, 64);">pyproject.toml</font>
    ，但可安装符合 PEP 621 标准的依赖。
    • 共性：
      • 所有工具都支持 <font style="color:rgb(64, 64, 64);">[project]</font> 中的 PEP 621 元数据（如 <font style="color:rgb(64, 64, 64);">name</font>, <font style="color:rgb(64, 64, 64);">version</font>, <font style="color:rgb(64, 64, 64);">dependencies</font>）。
      • 依赖解析逻辑类似（语义版本控制）。
    • 差异：
      • 依赖格式：Poetry 允许更灵活的版本语法（如 <font style="color:rgb(64, 64, 64);">^1.2.3</font>），PDM 使用 PEP 440 标准。
      • 锁定文件：Poetry 生成 <font style="color:rgb(64, 64, 64);">poetry.lock</font>，PDM 生成 <font style="color:rgb(64, 64, 64);">pdm.lock</font>，UV 支持 <font style="color:rgb(64, 64, 64);">requirements.txt</font>。
      • 插件系统：各工具可能有自定义配置（如 <font style="color:rgb(64, 64, 64);">[tool.poetry.scripts]</font>）。

    4. 学习路径建议¶

    • 从 PEP 621 开始：
      1. 阅读 PEP 621 文档，理解标准字段。
      2. 手动编写一个简单的 <font style="color:rgb(64, 64, 64);">pyproject.toml</font>（仅使用 <font style="color:rgb(64, 64, 64);">[project]</font> 和 <font style="color:rgb(64, 64, 64);">[build-system]</font>）。
    • 选择工具并实践：

    Poetry：适合需要严格依赖锁定的项目。

    Text Only
    1
2
    poetry init  # 生成 pyproject.toml
poetry add requests  # 添加依赖

    PDM：适合 PEP 621 原生支持的项目。

    Text Only
    1
2
    pdm init    # 生成 pyproject.toml
pdm add requests

    UV：适合替代 <font style="color:rgb(64, 64, 64);">pip</font> 提升安装速度。

    TOML
    1
      uv pip install -r requirements.txt  # 兼容 pip 命令
    • 对比迁移：
      • 尝试将一个项目的配置从 Poetry 迁移到 PDM，观察哪些字段需要修改。
      • 使用 <font style="color:rgb(64, 64, 64);">uv</font> 安装基于不同工具的依赖，测试兼容性。

    5. 关键注意事项¶

    • 工具锁文件不互通：<font style="color:rgb(64, 64, 64);">poetry.lock</font> 和 <font style="color:rgb(64, 64, 64);">pdm.lock</font> 格式不同，不能混用。
    • 构建系统差异：如果使用 <font style="color:rgb(64, 64, 64);">setuptools</font>、<font style="color:rgb(64, 64, 64);">flit</font> 或 <font style="color:rgb(64, 64, 64);">hatch</font>，需配置对应的 <font style="color:rgb(64, 64, 64);">build-backend</font>。
    • 动态字段：某些字段（如 <font style="color:rgb(64, 64, 64);">version</font>）可以通过工具动态生成（如 Poetry 的 <font style="color:rgb(64, 64, 64);">version = "0.0.0"</font> 配合 <font style="color:rgb(64, 64, 64);">poetry version</font> 命令）。

    6. 总结¶

    • 配置部分相通：所有工具在 PEP 621 标准部分（<font style="color:rgb(64, 64, 64);">[project]</font>）是兼容的。
    • 工具扩展不互通：如 <font style="color:rgb(64, 64, 64);">[tool.poetry]</font> 和 <font style="color:rgb(64, 64, 64);">[tool.pdm]</font> 是各自专用的。
    • 学习核心逻辑后，工具可快速切换：理解 PEP 621 后，只需熟悉各工具的扩展语法即可。
